## Layout — Web App

**App shell structure (D-14, locked):**

```
┌─────────────────────────────────────────────────────┐
│  VAppBar (64px tall)                                │
│  Left: EchoMind wordmark  Right: —                  │
├──────────────────┬──────────────────────────────────┤
│  Input pane      │  Results pane                    │
│  (left, 360px    │  (right, flex-grow 1)            │
│   fixed width)   │                                  │
```

- VAppBar height: 64px (Vuetify default — do not override)

**Primary visual anchor:** the Validate VBtn — full-width (`block`), large size, accent fill — the sole active element in the input pane before validation. Persona VSelect and PRD VTextField are visible but disabled, deliberately recessed so the eye lands on Validate.

## Spacing Scale

Declared values (multiples of 4 only). Vuetify's spacing scale uses multiples of 4px where `n` maps to `n*4px` (e.g. `ma-4` = 16px).

| Token | Value | Vuetify Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `*-1` | Icon gaps, chip padding |
| sm | 8px | `*-2` | Compact element spacing, button icon gap |
| md | 16px | `*-4` | Default element spacing, card internal padding |
| lg | 24px | `*-6` | Section padding, pane internal padding |
| xl | 32px | `*-8` | Layout gaps between panes |
| 2xl | 48px | `*-12` | Major section breaks |
| 3xl | 64px | `*-16` | Page-level top padding |

Exceptions:
- VAppBar height: 64px (Vuetify default, do not override with spacing tokens)
- Touch targets on Validate button: minimum 44px height (accessibility — use `size="large"` on VBtn)

## Typography

Vuetify 3 Material You type scale. Use semantic typography classes, not raw font-size overrides.

| Role | Size | Weight | Line Height | Vuetify Class | Usage |
|------|------|--------|-------------|---------------|-------|
| Body | 14px | 400 | 1.5 | `text-body-2` | Finding bullets, helper text, labels |
| Body Large | 16px | 400 | 1.5 | `text-body-1` | Section descriptions, PRD title display |
| Heading | 20px | 600 | 1.2 | `text-h6` | Section card titles (Fit, Friction, Questions, Refinements) |
| Display | 24px | 600 | 1.2 | `text-h5` | App bar wordmark, page-level heading |

Font: Roboto (Vuetify 3 default — no custom font loading needed).

Weights used: 400 (regular) and 600 (semibold). No other weights.

## Color

Vuetify 3 uses a Material You color system. Declare roles using Vuetify theme tokens, not raw hex.

| Role | Vuetify Token | Hex Equivalent | Usage |
|------|---------------|---------------|-------|
| Dominant (60%) | `surface` | `#FAFAFA` (grey-lighten-5) | Page background, pane backgrounds |
| Secondary (30%) | `surface-variant` | `#F5F5F5` (grey-lighten-4) | Cards, input pane background, VSelect background |
| Accent (10%) | `primary` | `#1565C0` (blue-darken-3) | Validate button, stepper active step, checked step icon |
| Destructive | not used in Phase 1 | n/a | No destructive actions in this phase |

**Accent reserved for (explicit):**
1. The "Validate" VBtn (filled, primary color)
2. The active step indicator on the VStepperItem (active color)
3. Completed step checkmark icon on the VStepper (success color, use Vuetify `color="success"` on completed items)

Accent is NOT used on: section card headers, persona name, PRD title, body text, disabled inputs.

**Disabled state:** `disabled` prop on VSelect and VTextField. Vuetify renders these with `opacity: 0.38` automatically — do not override.

## Component Inventory

Exact Vuetify components to use. Planner must not substitute alternate components without updating this spec.

### Input Pane

| Component | Vuetify | Configuration |
|-----------|---------|---------------|
| Persona selector | `VSelect` | `disabled`, `:items="['General Manager']"`, `label="Persona"`, `prepend-inner-icon="mdi-account"` |
| PRD title display | `VTextField` | `disabled`, `:model-value="'ACV MAX Auctions Integration'"`, `label="PRD"`, `prepend-inner-icon="mdi-file-document"` |
| Validate button | `VBtn` | `color="primary"`, `size="large"`, `block`, text: "Validate" |
| Input pane container | `VCard` | `variant="outlined"`, no elevation, `rounded="lg"` |

### Progress Indicator (D-15)

Use `VStepper` with `VStepperItem` for each stage. Render as a vertical stepper in the results pane while validation is in progress.

| Step | Label | Icon (complete) | State logic |
|------|-------|-----------------|-------------|
| 1 | Loading persona | `mdi-check-circle` | complete when validator called |
| 2 | Loading PRD | `mdi-check-circle` | complete when validator called |
| 3 | Calling validator | `mdi-check-circle` | complete when response received |
| 4 | Rendering output | `mdi-check-circle` | complete when sections render |

VStepper configuration: `non-linear`, `flat`, no edit affordance. Steps 1–2 complete immediately on click (they are instant for demo). Step 3 remains active during the gateway call. Step 4 completes as sections mount.

### Results Pane — Output Sections

Four `VCard` components rendered in a `VRow` + `VCol` grid (2-column on md+, 1-column on sm):

| Section | Card title | Icon |
|---------|-----------|------|
| Fit | "Fit" | `mdi-check-bold` |
| Friction | "Friction" | `mdi-alert` |
| Questions | "Questions" | `mdi-comment-question` |
| Refinements | "Refinements" | `mdi-pencil` |

Each card: `VCard` with `VCardTitle` (heading typography), `VCardText` containing a `VList` with `VListItem` per finding bullet. Card variant: `"elevated"`, elevation: `2`.

### App Bar

`VAppBar` with `elevation="1"`. Left: app name "EchoMind" in `text-h5` weight 600. No right-side controls in Phase 1.

## Interaction States — Web App

| State | What User Sees | Implementation |
|-------|---------------|----------------|
| Initial load | Input pane: persona + PRD pre-populated (disabled). Results pane: empty, shows empty state copy. Validate button: enabled. | On mount, no validation triggered |
| Validating | Validate button: `loading` prop (Vuetify spinner). Results pane: VStepper visible, progressing through steps. | `isValidating` reactive boolean |
| Results ready | VStepper hidden. Four section cards render. Validate button: re-enabled. | `results` reactive object populated |
| Demo-day replay mode | Identical to Results ready but instant (no stepper shown > 500ms). | `VITE_REPLAY_MODE=true` skips real gateway call |

No error states in Phase 1 (LLM-03 is Phase 2). If the gateway call fails during the demo, the demo-day fixture replay (D-08) is the mitigation — not a structured error UI.

## Skill Surface — Markdown Contract (D-17)

The skill surface renders CommonMark markdown in Claude Code's terminal. No Vuetify. No CSS.

**Required heading hierarchy:**

```markdown
# Validation: General Manager → ACV MAX Auctions PRD

## Fit

- [finding bullet]
- [finding bullet]

## Friction

- [finding bullet]
- [finding bullet]

## Questions

- [finding bullet]
- [finding bullet]

## Refinements

- [finding bullet]
- [finding bullet]
```

Rules:
- `#` H1: exactly one per output, format `Validation: {persona name} → {PRD title}`
- `##` H2: exactly four sections in order: Fit, Friction, Questions, Refinements
- Each finding is a `-` list item (single dash, not `*` or `+`)
- No bold, no italic, no nested lists in Phase 1 output
- No trailing whitespace; blank line between each section
