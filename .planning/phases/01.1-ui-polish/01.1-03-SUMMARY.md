## Decisions Made
- InputPane card uses `variant="flat"` with explicit inline `style` rather than a Vuetify surface prop, because Vuetify's `color` prop on v-card controls text/icon color not background — direct CSS var reference is the reliable path for container fills
- The negation grep `! grep "variant=\"outlined\""` in the plan's verification command was imprecise: the VSelect and VTextField inside InputPane intentionally retain `variant="outlined"` for their form controls. The v-card itself correctly uses `variant="flat"`. This is not a deviation — the plan acceptance criteria explicitly scoped it to the VCard.
