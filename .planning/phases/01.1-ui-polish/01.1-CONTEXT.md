# Phase 1.1: UI Polish — Context

**Gathered:** 2026-05-05
**Status:** Ready for planning
**Source:** Direct session context (Chad + Claude, Phase 1 execution session)

<domain>
## Phase Boundary

This phase delivers visual polish on the existing Vue 3 + Vuetify web app — no new features, no engine changes. The goal is the app looks intentional and on-brand before the recording. Three concerns: brand color system, logo header treatment, and component-level polish informed by reference design patterns.

Work is entirely in `apps/web/`. Engine, CLI skill, and planning artifacts are out of scope.

</domain>

<decisions>
## Implementation Decisions

### Material 3 Theme
- A Material Theme Builder export exists at `assets/material-theme-builder/material-theme.json`
- Seed color: `#9030A8` (the logo purple). Light scheme primary: `#7A4F81`
- The current app uses a hardcoded `primary: '#1565C0'` (blue) — this must be replaced
- Apply the light scheme from the JSON to Vuetify's `createVuetify()` in `apps/web/src/main.ts`
- Map at minimum: `primary`, `surface`, `surfaceVariant`, `onSurface`, `error`, `secondary`
- Dark mode is a nice-to-have, not required for Phase 1.1
- The M3 JSON is the source of truth — do not hand-pick colors, read them from the file

### Logo Header
- Logo PNG: `assets/logo/logo-echomind.png` (also copied to `apps/web/public/logo-echomind.png`)
- Current state: `<img height="40" object-fit="cover" object-position="center top">` — clips the tagline text "VIRTUAL USER TESTING" but the icon is small and hard to read on camera
- The PNG has a white/light background — on a white `surface` app bar this creates a subtle white square
- Options discussed: tighter crop PNG, transparent bg, or use `object-position` to show just the icon head
- Chad will optimize the PNG himself if needed — the implementation should be flexible to accept an updated asset with no code change
- For the recording, the logo should be readable at standard screen recording resolution (1080p or 4K)
- Desired: icon-only visible (the head/brain mark), no "ECHOMIND VIRTUAL USER TESTING" text below it, no white square artifact

### Component Polish
- Reference visual direction: clean, data-dense tool aesthetic (think Linear, Notion, Vercel dashboard)
- Section cards currently have thin separator dividers between findings — this is OK but could be improved
- The "Configure" pane card (left panel) uses `variant="outlined"` — should feel like a sidebar, not a form
- The `GET FEEDBACK` button is the primary CTA — should feel confident, not generic Vuetify default
- Empty state icon (`mdi-lightning-bolt-outline`) + "Ready for feedback" text — currently functional, could be more on-brand
- Typography: Roboto is fine, but font weights and size hierarchy could be tightened
- No new interaction patterns — this is visual CSS/theme polish only

### What NOT to do
- Do not add new components or routes
- Do not touch `packages/engine/` or `apps/skill/`
- Do not implement dark mode toggle (nice-to-have, not required)
- Do not change the layout (360px input pane + flex results pane is correct)
- Do not change copy (button labels, section titles already approved)

### Claude's Discretion
- Exact Vuetify color role mapping (which M3 roles map to which Vuetify keys)
- Whether to use CSS custom properties or Vuetify theme exclusively
- Specific border-radius, elevation, and spacing values
- Whether to add subtle hover states to section finding rows

</decisions>
